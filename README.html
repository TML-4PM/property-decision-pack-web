Property Decision Pack - User Guide

Purpose: This app helps Troy and Cassandra compare property sale and retention options using data-driven financial modelling.

How to Use This App

  1. Review key numbers - Start by understanding current property values, debts, and net proceeds from each sale option
  2. Compare options - Look at the options overview to see what each path means for debt, cash, and friction
  3. Run stress tests - Check how each option performs under adverse conditions (job loss, reduced income)
  4. Explore projections - See property value growth over 5-10 years under different growth assumptions
  5. Score decisions together - Use the decision matrix to rate each option on multiple criteria
  6. Log questions - Record questions and agreed answers as you work through scenarios
Tip: Work through the app top-to-bottom in your first session. Then use the question log for follow-up discussions.

Understanding the Sections

Key Numbers

Shows the current state of both properties:

  • Market values for Seabrook and Peninsular
  • Current debt on Seabrook
  • Net cash after selling costs (2.5% + $6k fixed per property)

Options Overview

Compares three main scenarios:

  • Sell Seabrook, keep Peninsular - Clears debt, leaves $640k buffer, but smaller property
  • Sell Peninsular, keep Seabrook - Keeps the larger home, minimal debt, but tight cashflow
  • Sell both - Maximum flexibility, $1.4M cash, zero debt, but requires renting

Stress Test

Models "hit point" scenarios where income drops significantly:

  • Weekly cashflow (mortgage + rent income - family rent)
  • Runway weeks from available buffer cash
  • Safety margin before severe financial stress
Important: These are modelling scenarios only, not predictions. Actual outcomes will vary.

Growth Projections

Property value forecasts under three growth rates:

  • Low (3%): Conservative scenario
  • Middle (5%): Moderate long-term average
  • High (7%): Optimistic scenario

Click the scenario chips to switch between them.

Decision Matrix

Score each option on five criteria (1-5 scale):

  • Cashflow safety: Can you survive income shocks?
  • Staying housed: Do you keep a stable home?
  • Staying in market: Do you maintain property exposure?
  • Fairness: Does the split feel equitable?
  • Friction: How much disruption does this cause? (lower is better)
Tip: Score these together. Disagreements are valuable data, not problems.

Question Log

Use this to capture:

  • Questions that come up during discussion
  • Answers you agree on
  • Which assumptions or numbers informed each answer

Important: Question log is saved locally in your browser. Use "Export log json" to save a permanent copy.

Updating the Data

To change assumptions or model inputs:

  1. Open model_data.json in a text editor
  2. Find the relevant section:
    • assumptions - Selling costs, move friction, etc.
    • key_numbers - Property values and debts
    • options_recap - Scenario outcomes
    • stress_inputs - Stress test parameters
  3. Edit the values
  4. Save the file
  5. Refresh your browser
Warning: Export your question log before updating data. Treat each data change as a new version.

Best Practices

  • Export your question log after important discussions
  • Version your data - Save copies of model_data.json when you make major changes
  • Score together - The decision matrix is most valuable when both people participate
  • Use concrete examples - When discussing scenarios, reference specific numbers from the tables
  • Challenge assumptions - The assumptions table exists precisely so you can question and revise inputs

Limitations

  • This is a modelling tool, not financial advice
  • Projections are illustrative scenarios, not predictions
  • Does not account for taxes, capital gains, or legal costs beyond selling fees
  • Stress tests use simplified weekly cashflow models
  • Property value forecasts assume steady growth (reality is volatile)
Recommended: Use this app to structure your thinking and discussions, then consult with a qualified financial advisor before making final decisions.

Support

For technical issues with the app:

  • Check that model_data.json is in the same folder as index.html
  • Ensure chart images are in the assets/ subfolder
  • Try opening in a different browser (Chrome, Firefox, Safari)
  • Check the browser console for errors (right-click → Inspect → Console)
